    How the habit program works

    Default habits use seven Sunday–Saturday calendar weeks in Phase 1. Week 1 is the week that contains your program start; week k needs k logged days anywhere that week (only on/after your start date counts). Time habits ramp Phase 1 from 1 minute on day 1 up to n minutes on day n (you set n when creating the habit). After Phase 1, two Phase 2 models run in parallel: Progress (accent bar) milestones 8→90 can be “packed” into your latest contiguous streak; any 90-day streak in Phase 2 completes the program on that path. Time habits hold n minutes per active Phase 2 day. Long run (green bar) is the original ladder with required rest days between streaks — same rules the day planner deadlines use for streak requirements. Worknight (settings toggle) counts Sun–Thu only; Fri/Sat are ignored. Double-tap a calendar day for a cheat freeze (max 2 per 30 days). In details, each long-run leg is labeled L8, L9, … L90: the number is how many consecutive days (or Sun–Thu work nights) that segment needs before the next mandatory blank day. The calendar shows nominal rest days from your Phase 2 start as a guide when following the ladder exactly; effective rests (from your logs + planner hints) override how cells are highlighted when they differ. Overall complete when either path reaches 100%. Default habits total 4095 points; Time habits use a dynamic Phase 1 cap plus 4067 Phase 2 points. Your habits are written to IndexedDB after every change and mirrored to localStorage for recovery. Data stay on this device only.
