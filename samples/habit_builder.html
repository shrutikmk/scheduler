    Phase 1 uses seven Sunday–Saturday calendar weeks. Week 1 is the week that contains your program start; week k needs k logged days anywhere that week (only on/after your start date counts). After phase 1, streaks of 8…90 days with one rest day after each streak except after the final 90-day streak. Missing a habit day resets the current streak only (finished streaks stay counted). Highlighted empty days are required rest — leave them unchecked. 100% = 4095 points. Your habits are written to the browser’s IndexedDB database (scheduler-habit-builder-db) after every change (toggle day, start date, add/delete habit, switch selection). The same snapshot is mirrored to localStorage for recovery if IndexedDB fails. Data stay on this device only — clearing site data removes them. There is no cloud backup. Use Export backup to download JSON (save it under scheduler/backups/ in the repo if you like). Use Import backup to restore from that file; the browser cannot write into the repo by itself. You can toggle any day on or after program start (including future calendar days). You can track multiple habits — each has its own start date, calendar, and progress bar.
